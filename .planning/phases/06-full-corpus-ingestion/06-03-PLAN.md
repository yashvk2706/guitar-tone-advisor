<objective>
Implement the YouTube transcript loader and chunker. This plan delivers INGEST-10: load_youtube_transcripts() reads raw_data/youtube_ids.txt (13 video IDs with inline comments), fetches each transcript via the v1.x instance API, stores raw segments in RawDocument.metadata for start_time threading, and chunk_youtube() packs them into 300-500 token windows. The yt-dlp subprocess fallback handles geo-blocked or restricted videos.

Purpose: YouTube transcripts (Mark Knopfler tone, SRV tone, Eddie Van Halen gear, funk rhythm) directly address 3 of the 5 held-out eval topics. They are the second-most-important source after PDFs for hitting recall@8 >= 1.0.
Output: load_youtube_transcripts() + _parse_youtube_ids() + _load_via_ytdlp() in loader.py; chunk_youtube() in chunker.py; updated dispatch; unit tests for both.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| youtube_ids.txt | Committed file of 13 video IDs — treated as trusted input from developer |
| yt-dlp subprocess | video_id validated as 11-char [A-Za-z0-9_-] before passing to subprocess list (shell=False) |
| YouTube transcript content | Treated as untrusted text — stored verbatim, no code execution path |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-06 | Tampering | yt-dlp shell injection via video_id | mitigate | validate video_id with re.match(r'^[A-Za-z0-9_-]{11}$', video_id) before subprocess call; use list form subprocess.run([...], shell=False) — never string form |
| T-06-07 | Denial of Service | youtube-transcript-api network hang | accept | Single-user local tool; API calls are at ingest time only; network timeout from OS applies |
| T-06-08 | Spoofing | "youtube_transcript" source_type bypassing DB CHECK | mitigate | source_type="youtube" hardcoded in load_youtube_transcripts(); test_youtube_source_type_not_transcript verifies the string "youtube_transcript" never appears; DB CHECK raises on violation |
</threat_model>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user query string → expand_query → embed_query | Untrusted text from chat layer traverses alias expansion then embedding API |
| embed_query result → psycopg3 cursor | 1536-d list[float] enters SQL params tuple via %s placeholder |
| psycopg3 cursor → chunks table | Parameterized query; no user string ever interpolated into SQL |
| chunks table → ChunkResult | DB row deserialized into frozen typed dataclass; metadata_json.get() with default prevents KeyError |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | _RETRIEVE_SQL execution | mitigate | All parameters via %s placeholders; no f-string SQL enforced by test_no_fstring_sql static scan in tests/test_retrieval.py |
| T-02-02 | Tampering | Direct openai import bypass | mitigate | retrieval code routes through get_embedder() only; test_no_direct_openai_import scans app/retrieval/*.py and fails if any import openai statement is found |
| T-02-03 | Tampering | register_vector called in retrieve() | mitigate | test_register_vector_not_in_retrieve asserts "register_vector" string does not appear in base.py; registration is exclusively get_conn()'s responsibility |
| T-02-04 | Information Disclosure | metadata_json deserialization | accept | metadata_json is written by the project's own ingest pipeline; no untrusted external content; .get("source_filename", "unknown") prevents KeyError leakage |
| T-02-05 | Denial of Service | embedding_model filter returns 0 rows | accept | Correct behavior when model mismatch; empty list returned to caller; EXPLAIN ANALYZE log in debug mode surfaces the issue quickly |

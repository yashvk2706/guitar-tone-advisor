<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → POST /chat | Untrusted JSON body with session_id, message, gear |
| client → GET /sources/{chunk_id} | Untrusted path parameter chunk_id (potential SQL injection vector) |
| FastAPI → Anthropic API | ANTHROPIC_API_KEY flows from Settings to AsyncAnthropic — must not be logged |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-07 | Tampering | GET /sources/{chunk_id} — SQL injection via path param | mitigate | _SOURCES_SQL uses %s::uuid placeholder — psycopg parameterization + PostgreSQL UUID cast rejects any non-UUID string (e.g., "'; DROP TABLE chunks; --" fails UUID cast, returns 404); no f-string SQL (enforced by test_no_fstring_sql_in_main) |
| T-03-08 | Information Disclosure | POST /chat error handler — ANTHROPIC_API_KEY leakage | mitigate | Use logger.error("%r", e) (repr, not traceback.format_exc()); HTTPException(status_code=500) returns generic message to client; API key value never appears in response body or logs |
| T-03-09 | Denial of Service | POST /chat — unbounded message body | accept | Pydantic validates ChatRequest model; FastAPI default max body size is sufficient for a single-user local tool; no file uploads involved |
| T-03-10 | Tampering | POST /chat — gear block injection into first user message | accept | Gear content is injected as a <gear> JSON block into the user message; the model's grounding rules (D-13) treat it as context, not as a command override; acceptable risk for personal local tool |
</threat_model>

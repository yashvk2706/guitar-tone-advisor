<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user message → system prompt | User content injected into <sources> XML block and user message turn — model-readable but not model-executable |
| ANTHROPIC_API_KEY → generator | API key flows from Settings to AsyncAnthropic constructor — must never appear in logs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | app/generation/prompt.py — <sources> XML injection | accept | <sources> XML is model-readable context, not executable code; the model's grounding rules (D-13) prevent fabrication; post-stream validation (D-08) discards out-of-range citations. System prompt is stable and cannot be overridden by user content. Acceptable risk for personal local tool. |
| T-03-02 | Information Disclosure | app/generation/generator.py — ANTHROPIC_API_KEY in error logs | mitigate | Use repr(e) not traceback.format_exc() in any exception handler (per Phase 1 Plan 04 established pattern). AsyncAnthropic constructed with explicit api_key=get_settings().anthropic_api_key — key value never logged. |
| T-03-03 | Tampering | Citation regex — [Sn] references injected by model | mitigate | Post-stream validator (D-08/D-09) discards any n where n > len(sources). Model cannot reference a source that was not injected into <sources> XML. Citations are validated before the event: citations payload is emitted. |
</threat_model>

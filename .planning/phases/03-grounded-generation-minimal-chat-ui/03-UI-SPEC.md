# Phase 3 — UI Design Contract

> Visual and interaction contract for the minimal Next.js chat UI.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker before implementation.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none — Tailwind utilities only (D-01) |
| Icon library | lucide-react (standard for Next.js; zero-config tree-shaking) |
| Font | `font-mono` for citations and chunk text (Tailwind default: ui-monospace, SFMono-Regular, Menlo, Monaco); `font-sans` for conversation prose (Tailwind default: ui-sans-serif, system-ui, -apple-system) |

**Rationale:** D-01 locks Tailwind-only, no CSS Modules for Phase 3. The developer-tool aesthetic (dark terminal, high readability) is achieved entirely through Tailwind utility classes. No third-party registry vetting is required. Phase 4 may revisit CSS Modules for rotary-knob SVGs.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Tailwind Class | Value | Usage |
|-------|---------------|-------|-------|
| xs | `p-1` / `gap-1` | 4px | Icon gaps, citation pill internal padding, cursor dot |
| sm | `p-2` / `gap-2` | 8px | Source-type label padding, compact badge spacing |
| md | `p-4` / `gap-4` | 16px | Message bubble padding, input field padding, drawer header padding |
| lg | `p-6` / `gap-6` | 24px | Inter-message vertical spacing, section separators |
| xl | `p-8` / `gap-8` | 32px | Input area top/bottom padding within the sticky bar |
| 2xl | `p-12` | 48px | Empty state vertical centering offset |
| 3xl | `p-16` | 64px | Page-level horizontal gutter on wide viewports |

Exceptions:
- Touch target for "New chat" button: minimum `h-9 px-4` (36px height) to ensure comfortable click target
- Citation pill height: `h-5` (20px) — intentionally compact to sit inline with text
- Input textarea minimum height: `min-h-[48px]` (48px) — comfortable single-line target before expansion

### 2. Message Bubbles

**User message:**
- Right-aligned: `flex justify-end`
- Bubble: `max-w-[70%] bg-zinc-800 rounded-2xl rounded-tr-sm px-4 py-3 text-sm leading-relaxed text-zinc-50`
- No avatar, no label

**Assistant message:**
- Left-aligned: `flex flex-col gap-2`
- Bubble: `max-w-[80%] bg-zinc-900 rounded-2xl rounded-tl-sm px-4 py-3 text-sm leading-relaxed text-zinc-50`
- Below the bubble: citation pill row + coverage indicator (only rendered after `event: citations` fires)

Inter-message gap: `space-y-6` on the container (24px).

---

### 3. Streaming Token Render

**Before any tokens arrive (post-submit, awaiting first token):**
- Show an empty assistant bubble with a single animated cursor: three-dot pulse or a blinking underscore
- Tailwind: `after:content-['▋'] after:animate-pulse after:text-zinc-400`
- The bubble is visible immediately after submit so the user sees something in < 100ms

**During streaming:**
- Tokens are appended to the same assistant bubble in real time — no re-render flicker
- A `▋` cursor character is appended to the current text content; it is removed when the stream ends
- The cursor class: `text-zinc-400 animate-pulse`
- Message list auto-scrolls to bottom as tokens arrive: call `scrollIntoView({ behavior: "smooth" })` on the last message element after each token append

**After stream completes:**
- Cursor is removed
- Citation pills appear (see §5 below — they are NOT shown during streaming)
- Coverage indicator appears below the bubble

---

### 4. Citation Pills

Citation pills render **only after `event: citations` fires** — they are absent from the DOM during streaming. This is a hard interaction rule from D-08.

**Pill appearance:**
```
[S1]
```
- Inline element rendered within a `flex flex-wrap gap-1 mt-2` container below the assistant bubble (not inline within the prose text in Phase 3)
- Tailwind: `inline-flex items-center h-5 px-1.5 rounded text-xs font-medium font-mono bg-blue-950 text-blue-400 border border-blue-800 cursor-pointer hover:bg-blue-900 hover:text-blue-300 hover:border-blue-600 transition-colors`
- Pills are ordered S1, S2, ... Sn in the order they appear in the `event: citations` payload
- An invalid `[Sn]` reference that was NOT in the citations payload is simply not rendered — no non-interactive badge, no broken indicator

**Pill interaction:**
- `onClick`: open the citation drawer for this pill's `chunk_id` (lazy fetch via `GET /sources/{chunk_id}`)
- `onMouseEnter`: no tooltip in Phase 3 — drawer is the only way to see chunk text
- Active/focused pill: `ring-1 ring-blue-500 ring-offset-1 ring-offset-zinc-950`

### 5. Citation Drawer

**Trigger:** click a citation pill.

**Placement:** right-side drawer sliding in from the right edge of the viewport. In Phase 3, a simple overlay is acceptable: fixed-position panel anchored to the right side.

**Drawer dimensions:** `w-80 md:w-96` (320px / 384px), full height (`h-full`), `bg-zinc-900 border-l border-zinc-700`

**Drawer layout:**
```
┌──────────────────────────────────────┐
│  [Forum]  bb_king_tone.txt      [✕]  │  ← header row: source-type badge + name + close
├──────────────────────────────────────┤
│  Section / source name               │  ← secondary label (source_name, muted)
├──────────────────────────────────────┤
│                                      │
│  ┌───────────────────────────────┐   │
│  │ raw chunk text block          │   │  ← scrollable, monospace, zinc-800 bg
│  │ font-mono text-xs leading-    │   │
│  │ relaxed text-zinc-300         │   │
│  └───────────────────────────────┘   │
│                                      │
└──────────────────────────────────────┘
```

**Drawer header row:** `flex items-center justify-between px-4 py-3 border-b border-zinc-700`

**Source-type badge** (CITE-02):
- `[Forum]` → `bg-emerald-950 text-emerald-400 border border-emerald-800`
- `[Manual]` → `bg-amber-950 text-amber-400 border border-amber-800`
- `[Article]` → `bg-violet-950 text-violet-400 border border-violet-800`
- `[YouTube]` → `bg-rose-950 text-rose-400 border border-rose-800`
- Common: `inline-flex items-center h-5 px-1.5 rounded text-xs font-medium font-mono`

**Source name label:** `text-xs text-zinc-400 font-normal truncate flex-1 mx-2`

**Close button:** `h-6 w-6 flex items-center justify-center text-zinc-400 hover:text-zinc-50 rounded hover:bg-zinc-700 transition-colors` — use lucide-react `X` icon (16px)

**Chunk text block:** `mx-4 mb-4 p-3 bg-zinc-800 rounded-lg text-xs font-mono leading-relaxed text-zinc-300 overflow-y-auto max-h-[calc(100vh-8rem)]`

**Loading state** (between pill click and `GET /sources/{chunk_id}` response):
- Show a single-line skeleton: `h-4 bg-zinc-700 rounded animate-pulse mx-4 my-3`
- Three skeleton lines stacked to hint at content

**Error state in drawer:**
- `text-xs text-red-400 font-mono mx-4 my-3` — "Failed to load source. Try again."

**Close behavior:**
- Click the X button
- Click the backdrop overlay (`fixed inset-0 bg-black/40 z-40`) to dismiss
- No keyboard shortcut in Phase 3

**Backdrop:** `fixed inset-0 bg-black/40 z-40` rendered behind the drawer panel when drawer is open; clicking it closes the drawer.

**Drawer z-index:** `z-50` (above page content, below any future modals)

### 6. Coverage Indicator

**Placement:** directly below the assistant bubble, above the citation pill row.

**Copy format:** `● N sources agree` where N is `len(validated_citations)` from `event: citations`.

**Tailwind:** `flex items-center gap-1.5 text-xs text-zinc-400 mt-1`

**Dot:** `h-2 w-2 rounded-full bg-green-500 shrink-0`

**Text:** `text-zinc-400` (muted — supporting info, not primary content)

**N=1 case:** render as `● 1 source` (no "agree" — singular).
**N=0 case:** do not render the coverage indicator at all.

---

### 7. "New Chat" Button

**Placement:** top-right of the header bar.

**Visual:** `h-8 px-3 rounded-md text-xs font-medium bg-zinc-800 text-zinc-300 border border-zinc-700 hover:bg-zinc-700 hover:text-zinc-50 transition-colors` — subdued, utility-style. Not an accent color.

**Label:** `New Chat` (lucide-react `SquarePen` icon at 14px before the text, optional for Phase 3)

**Interaction:** No confirmation dialog. Immediate reset.
- On click: set `sessionId` state to `null` in React
- Clear the `messages` array in React state
- Clear any open citation drawer
- The textarea clears and regains focus
- Rationale: this is a personal single-user tool; the cost of accidental reset is negligible and a confirmation dialog adds friction for every deliberate reset. D-17 confirms no backend DELETE endpoint is needed.

### 8. Empty State

**Shown when:** the message list is empty (new session, or after "New Chat" is clicked).

**Layout:** centered vertically and horizontally in the message list area: `flex flex-col items-center justify-center h-full text-center px-8 gap-4`

**Content:**
```
[guitar icon — lucide-react Guitar, 32px, text-zinc-600]

Start with your gear

Describe what you're playing (guitar, amp, pedals)
and the tone you're chasing — I'll cite sources
```

- Heading: `text-lg font-semibold text-zinc-400` (not zinc-50 — intentionally muted; this is ambient, not an alert)
- Body: `text-sm text-zinc-500 max-w-sm leading-relaxed`
- Icon: `text-zinc-600` (barely visible — decorative only)

### 9. Input Area

**Input element:** `<textarea>` (not `<input type="text">`), resizing vertically as the user types.

**Tailwind:** `w-full bg-zinc-800 border border-zinc-700 rounded-xl px-4 py-3 text-sm text-zinc-50 placeholder-zinc-500 resize-none leading-relaxed focus:outline-none focus:ring-1 focus:ring-blue-500 focus:border-blue-500 transition-colors`

**Initial height:** `rows={1}` with `min-h-[48px]`; grows dynamically via JavaScript `scrollHeight` assignment to a `max-h-[200px]` cap.

**Placeholder copy:** `Describe your gear and target tone…`

**Submit button:**
- Tailwind: `ml-2 h-10 w-10 flex items-center justify-center rounded-xl bg-blue-600 hover:bg-blue-500 active:bg-blue-700 text-white transition-colors disabled:opacity-40 disabled:cursor-not-allowed disabled:pointer-events-none`
- Icon: lucide-react `ArrowUp` (20px) — a common convention for "send" in modern chat UIs
- Label: visually hidden `sr-only` text "Send message" for accessibility

**Input bar layout:** `flex items-end gap-2`

**Submit behavior:**
- `Enter` (without Shift) submits the message and clears the textarea. This is the primary submit path.
- `Shift+Enter` inserts a newline — multi-line input.
- Submit button click also submits.

**Disabled state during streaming:**
- Textarea: `disabled` attribute + `opacity-50 cursor-not-allowed` via Tailwind
- Send button: `disabled` attribute (the `disabled:` Tailwind variant handles the visual)
- The user cannot submit a new message while a response is streaming. This is intentional for Phase 3 simplicity. Phase 4 may add a "Stop" button.

---

### 10. Error State

**Trigger:** SSE stream failure (network error, server 500, or timeout mid-stream).

**Placement:** An error bubble rendered in the message list at the position where the assistant response would have appeared — same left-aligned layout as an assistant bubble.

**Visual:**
```
<div class="max-w-[80%] bg-red-950 border border-red-800 rounded-2xl rounded-tl-sm px-4 py-3">
  <p class="text-xs font-medium text-red-400 mb-1">Response failed</p>
  <p class="text-sm text-red-300">The server didn't respond. Check that the FastAPI server is running on port 8000 and try again.</p>
</div>
```

**No retry button in Phase 3.** The user resubmits by typing again or pressing the up arrow to recover their message. (Phase 4 can add a retry button.)

**Network error detection:** if `fetch()` throws, or if the `ReadableStream` reader produces a non-200 status, render the error bubble and re-enable the input.

## Interaction Contracts

### SSE Stream Lifecycle (D-05, D-06, D-07)

```
User submits message
  → set isStreaming = true
  → disable textarea + send button
  → append user message bubble to list
  → append empty assistant bubble with ▋ cursor

fetch('/api/py/chat', { method: 'POST', body: JSON.stringify({session_id, message}) })
  → response.body.getReader() → ReadableStream parsing loop

  event: session  →  persist session_id in React state (useState)

  data: {"text": "<token>"}  →  append token to current assistant bubble text
                                 scroll message list to bottom

  [stream ends: reader.done = true]
    → remove ▋ cursor from assistant bubble
    → set isStreaming = false
    → re-enable textarea + send button

  event: citations  →  validate source IDs against the SSE payload
                        render citation pill row below the bubble
                        render coverage indicator
```

**SSE line parser (custom hook `useSSEStream`):**
- Accumulate chunks from the ReadableStream into a text buffer
- Split on `\n\n` to extract individual SSE events
- Parse `event:` line (if present) and `data:` line
- Handle `event: session`, plain `data:` (token), `event: citations`

**Session ID persistence:** `useState<string | null>(null)`. On `event: session`, call `setSessionId(payload.session_id)`. Subsequent submits send the stored value. On "New Chat", call `setSessionId(null)`.

### Citation Pill Reveal (D-08)

- Citation pills are rendered from the `event: citations` payload — which arrives AFTER the stream ends.
- The pills section is entirely absent from the DOM during streaming.
- The pills container uses a `citations` state variable initialized as `null`; it renders only when `citations !== null`.
- Pill click triggers `fetchSourceChunk(chunk_id)` which calls `GET /api/py/sources/{chunk_id}`.

### Citation Drawer Open/Close

- One drawer open at a time. Opening a second pill replaces the first drawer's content.
- Drawer state: `useState<{chunkId: string, data: SourceData | null} | null>(null)`.
- Open: set drawer state with `chunkId`, immediately show skeleton, fetch data, set data on response.
- Close: set drawer state to `null` (via X button or backdrop click).
- Drawer uses `fixed` positioning; no scroll lock required for the simple overlay approach.

### Enter-to-Submit

- `onKeyDown` on the textarea: if `e.key === 'Enter' && !e.shiftKey`, call `e.preventDefault()` and submit.
- During streaming (`isStreaming === true`), the `onKeyDown` handler returns early — no double-submit.

### "New Chat" Immediate Reset

```
onClick: () => {
  setSessionId(null);
  setMessages([]);
  setCitations(null);
  setDrawer(null);
  setIsStreaming(false);
  textareaRef.current?.focus();
}
```

No confirmation dialog. No backend call. The old session persists in FastAPI memory until server restart (acceptable per D-17).

### Auto-scroll Behavior

- After each token append: scroll the message list container to `scrollTop = scrollHeight`.
- Use a `useEffect` that watches the messages array for changes.
- Do not force-scroll if the user has manually scrolled up to read earlier messages (detect with `isScrolledToBottom` helper: `scrollTop + clientHeight >= scrollHeight - 20`).

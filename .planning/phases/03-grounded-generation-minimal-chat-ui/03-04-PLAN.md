<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → Next.js dev server | User input in textarea submitted via fetch POST |
| Next.js dev server → FastAPI | Proxy rewrite forwards /api/py/* to localhost:8000 — local only, no CORS config needed |
| FastAPI response → browser | SSE stream and JSON responses consumed by custom hook |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-11 | Tampering | frontend/hooks/useSSEStream.ts — malformed SSE frames | accept | JSON.parse() is wrapped in the event loop; malformed frames cause a caught exception that triggers onError callback and renders the error bubble; no silent failure |
| T-03-12 | Information Disclosure | frontend — session_id in React state | accept | session_id is a random UUID with no semantic value; it identifies a server-side dict entry only; no PII or credentials in client state; single-user local tool |
| T-03-13 | Denial of Service | frontend — unrestricted textarea input length | accept | Single-user local tool; no server-side message length enforcement needed in Phase 3; FastAPI default body size limit applies |
| T-03-14 | Tampering | frontend/components/CitationDrawer.tsx — chunk_text XSS | mitigate | chunk_text is rendered as {drawer.data.chunk_text} inside a React element (JSX), not via dangerouslySetInnerHTML; React escapes all text content by default; no raw HTML injection possible |
</threat_model>

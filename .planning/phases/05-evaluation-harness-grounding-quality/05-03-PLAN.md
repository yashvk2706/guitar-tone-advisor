<objective>
Build the custom RAGAS-style faithfulness CLI (`python -m app.eval.ragas`) that measures the hallucination rate of generated answers: for each held-out tuple it generates a live answer, decomposes the answer into atomic claims, checks each claim against the retrieved chunks, and logs `supported / total` faithfulness per query.

Purpose: Quantify how much of each answer is actually grounded in the corpus vs. invented — the hallucination metric that complements retrieval recall. Built entirely on the direct `anthropic` SDK (no `ragas` library, no LangChain, no HuggingFace) per D-04 / CLAUDE.md.
Output: `app/eval/ragas.py` (CLI + helpers), `tests/test_eval_ragas.py` (6 tests), `eval/faithfulness_runs.jsonl` (created on first run).
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| generated answer → claim extractor | LLM-generated text is fed into a second LLM prompt — prompt-injection surface |
| LLM JSON response → parser | Model output may be malformed or markdown-fenced; parser must degrade safely |
| ragas CLI → Postgres | Retrieval crosses to DB; delegated to retrieve() (parameterized) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-02 | Tampering (prompt injection in claim decomposer) | app/eval/ragas.py CLAIM_EXTRACT/SUPPORT prompts | mitigate | System/user role separation: answer + claim text interpolated ONLY into user-role messages; system prompt fixed and trusted; parse_claims returns [] on parse failure (suspicious-perfect-score signal) |
| T-05-08 | Spoofing (malformed JSON faking perfect faithfulness) | parse_claims / parse_support | mitigate | parse_claims returns [] and parse_support returns False on JSONDecodeError; total==0 → faithfulness 0.0, not 1.0 — a failed extraction cannot masquerade as perfect grounding |
| T-05-03 | Injection / abstraction leak | app/eval/ embeddings | mitigate | No direct openai import (get_embedder() only); static guard test_no_direct_openai_import scans app/eval/ rglob (CLAUDE.md hard constraint) |
| T-05-09 | Information disclosure | ANTHROPIC_API_KEY in errors | accept | Anthropic()/AsyncAnthropic() own credential handling; CLI does not dump request bodies; surfaces construction errors fail-fast |
</threat_model>

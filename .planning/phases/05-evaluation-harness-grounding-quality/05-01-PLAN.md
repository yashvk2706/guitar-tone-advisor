<objective>
Build the retrieval recall scorer CLI (`python -m app.eval.retrieval`) that scores the existing golden eval set against the live HNSW retrieval layer, reports recall@1/5/8 + MRR, appends each run to an append-only log, and prints a diff vs the previous run.

Purpose: Replace vibes-based retrieval tuning with an objective, repeatable metric. Every future retrieval/embedding change can be judged against the held-out 5-tuple contract locked in Phase 1 before any tuning occurred.
Output: `app/eval/retrieval.py` (scorer + CLI), `tests/test_eval_retrieval.py` (8 tests), `eval/runs.jsonl` (created on first run).
</objective>

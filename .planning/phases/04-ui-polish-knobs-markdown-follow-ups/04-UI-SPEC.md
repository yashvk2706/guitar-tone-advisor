## Spacing Scale

All values are Tailwind 4 utility classes mapped to the 8-point scale. Established by Phase 3 component code.

| Token | Value | Tailwind class | Usage |
|-------|-------|----------------|-------|
| xs | 4px | `gap-1`, `px-1.5` | Icon gaps, citation pill padding |
| sm | 8px | `gap-2`, `px-2` | Inline badge gaps, small padding |
| md | 16px | `px-4`, `gap-4` | Message bubble padding, layout padding |
| lg | 24px | `py-6`, `px-6` | Message list vertical padding |
| xl | 32px | `px-8` | Empty state horizontal padding |
| 2xl | 48px | `md:px-16` | Desktop message list side padding |
| 3xl | 64px | — | Not used in Phase 4 |

**Exceptions:**
- Header height: `h-12` (48px) — touch target standard, not spacing token
- Knob SVG canvas: 80px × 88px per knob (SVG viewBox `0 0 80 88`), gap between knobs `gap-4` (16px)
- Knob label zone below arc: 16px reserved at bottom of SVG canvas (text at y=82)
- Follow-up pill min-height: 28px (`h-7`) — consistent with CitationPill `h-5` family, slightly taller for button affordance
- Loading indicator bottom margin inside bubble: `mt-1` (4px) — matches CoverageIndicator spacing

## Component Contracts

### 1. ReactMarkdown integration in MessageBubble.tsx (D-01)

**File:** `frontend/components/MessageBubble.tsx`

**Change:** Replace `{message.content}` raw text render in the assistant branch with:

```tsx
<div className="space-y-1">
  <ReactMarkdown components={MARKDOWN_COMPONENTS}>{message.content}</ReactMarkdown>
</div>
```

Where `MARKDOWN_COMPONENTS` is a module-level const mapping element names to the Tailwind classes declared in the Typography section above.

**Streaming behavior:** `react-markdown` v10 accepts a partial string as children and re-renders on each token append. The streaming cursor (`after:content-['▋'] after:animate-pulse after:text-zinc-400`) moves to the wrapping div — attach `streamingClass` to the outer `<div className="max-w-[80%] bg-zinc-900 ...">` wrapper, not the inner content div.

**Hard constraint (T-03-14):** `react-markdown` in JSX mode never calls `innerHTML`. Confirmed safe. Do not pass `rehype-raw` or any plugin that enables raw HTML passthrough.

**Plugin policy:** No remark/rehype plugins beyond the built-in set in react-markdown v10. No `remark-gfm` unless confirmed needed — the corpus system prompt uses standard Markdown, not GitHub-flavored (no tables, no task lists expected in tone advice responses).

### 2. streamPhase loading state machine in ChatPage.tsx (D-02)

**File:** `frontend/components/ChatPage.tsx`

**New state:** `streamPhase: 'idle' | 'searching' | 'drafting' | 'done'`

**Transition table:**

| Event | From | To | UI effect |
|-------|------|----|-----------|
| User submits message | idle | searching | Show "Searching corpus..." in assistant bubble |
| First `onToken` fires | searching | drafting | Show "Drafting..." in assistant bubble |
| `onDone` fires | drafting | done | Clear indicator, reset to idle after 300ms |
| `onError` fires | any | idle | Show error bubble, no loading indicator |
| New Chat | any | idle | Reset immediately |

**Loading indicator placement:** Inside the assistant message bubble, rendered as a bottom row beneath the streaming text content. Only visible while `msg.isStreaming === true`. Disappears when `onDone` fires (which also sets `isStreaming: false`).

**Loading indicator markup (within MessageBubble or passed via prop):**

```tsx
{message.isStreaming && (
  <p className="text-xs text-zinc-500 mt-1 animate-pulse">
    {streamPhase === 'searching' ? 'Searching corpus...' : 'Drafting...'}
  </p>
)}
```

**Implementation note:** `streamPhase` lives in `ChatPage.tsx`. Pass it as a prop to `MessageBubble` only for the currently-streaming message, or pass the derived label string. The simplest approach: `ChatPage` passes `loadingLabel?: string` to `MessageBubble` alongside `isLatestAssistant`.

**`isStreaming` state retained:** The existing `isStreaming: boolean` in ChatPage remains for input-disabled logic. `streamPhase` is additive alongside it.

### 3. RotaryKnob.tsx SVG component (D-04)

**File:** `frontend/components/RotaryKnob.tsx` (new file)

**Props:**
```tsx
interface RotaryKnobProps {
  name: string;   // e.g. "Bass"
  value: number;  // 0.0 – 10.0
}
```

**SVG geometry:**

- ViewBox: `0 0 80 88` (80px wide, 88px tall — 80px circle area + 16px label zone at bottom)
- Circle center: `cx=40 cy=40`
- Circle radius: `r=30`
- Track arc: full sweep from 7 o'clock to 5 o'clock = 270° total
  - Start angle: −135° from 12 o'clock = 7 o'clock
  - End angle: +135° from 12 o'clock = 5 o'clock
- Value arc: start at −135°, sweep = `(value / 10) * 270°`
- Arc drawn via SVG `<path>` with `stroke-linecap="round"`, stroke-width 5, `fill="none"`
- Center indicator dot: `<circle cx=40 cy=40 r=3>` — confirms center visually
- Needle line: optional — omit for Phase 4 (arc fill is sufficient indicator)

**Angle → SVG coordinate formula:**

```
angleDeg = -135 + (value / 10) * 270
angleRad = (angleDeg - 90) * π / 180   // SVG 0° is 3 o'clock; subtract 90° to start from 12
x = 40 + 30 * cos(angleRad)
y = 40 + 30 * sin(angleRad)
```

**Full arc path (track):**

```
// Start point at -135° (7 o'clock), end point at +135° (5 o'clock)
// Large arc flag = 1 (sweep > 180°)
const startX = 40 + 30 * cos((-135 - 90) * π/180)  // ≈ 18.8, 61.2
const endX   = 40 + 30 * cos((+135 - 90) * π/180)  // ≈ 61.2, 61.2
d = `M ${startX} ${startY} A 30 30 0 1 1 ${endX} ${endY}`
```

**Value arc path (fill):** Same formula, sweep angle = `(value/10) * 270`. Use `largeArcFlag = sweepAngle > 180 ? 1 : 0`.

**Labels:**
- Value label: `<text x="40" y="58" textAnchor="middle" fontSize="14" fontWeight="600">` — e.g. "7" — rendered in `fill-zinc-300`
- Name label: `<text x="40" y="82" textAnchor="middle" fontSize="10" fontWeight="400">` — e.g. "Bass" — rendered in `fill-zinc-500`

**Colors (Tailwind via className on SVG elements):**
- Track path: `className="stroke-zinc-700"`
- Value path: `className="stroke-zinc-300"`
- Center dot: `className="fill-zinc-400 stroke-none"`
- Value text: `className="fill-zinc-300"`
- Name text: `className="fill-zinc-500"`

**Sizing:** Each knob renders at its natural SVG size (80×88). Parent container uses `flex flex-row flex-wrap gap-4` to lay out multiple knobs horizontally. Max visible per row: 5 knobs at full width on mobile; wraps naturally.

### 4. parseKnobs.ts utility (D-03)

**File:** `frontend/utils/parseKnobs.ts` (new file)

**Signature:**
```ts
export interface KnobValue {
  name: string;   // e.g. "Bass" (capitalized as matched)
  value: number;  // 0–10 float, validated in range
}
export function parseKnobs(text: string): KnobValue[]
```

**Regex (from CONTEXT.md §Specific Ideas):**
```ts
const KNOB_RE = /\b(Bass|Mid|Treble|Gain|Volume|Presence|Reverb|Tone|Drive|Level|Sustain|Output|Bright|High|Low|Delay|Mix|Rate|Depth|Feedback)\s*[=:]\s*(\d+(?:\.\d+)?)/gi;
```

**Behavior:**
- Case-insensitive match (`i` flag)
- Global scan (`g` flag) — all occurrences in the completed message
- Last-value-wins for duplicate knob names (iterate all matches, overwrite same-name entry)
- Only return knobs where `parseFloat(value) >= 0 && parseFloat(value) <= 10`
- Return `[]` if no matches
- Operates on the **completed** message string only — never called during streaming

**Knob names list (exact 20):** Bass, Mid, Treble, Gain, Volume, Presence, Reverb, Tone, Drive, Level, Sustain, Output, Bright, High, Low, Delay, Mix, Rate, Depth, Feedback

**Render condition:** Same gate as citation pills — render only after `message.citations !== undefined && !message.isStreaming`.

### 7. Copy-to-clipboard button (D-05 discretion: per-message, full text)

**Placement:** Top-right corner of the assistant bubble content div, positioned absolutely. The bubble wrapper gets `relative` class added.

**Button:** Ghost button, icon-only (`Copy` from lucide-react, size 14).

**State:** Local `copied: boolean` state, resets to `false` after 2000ms via `setTimeout`.

**Markup:**
```tsx
<div className="relative group max-w-[80%] bg-zinc-900 rounded-2xl rounded-tl-sm px-4 py-3">
  <button
    onClick={handleCopy}
    className="absolute top-2 right-2 opacity-0 group-hover:opacity-100 transition-opacity h-6 w-6 flex items-center justify-center rounded text-zinc-700 hover:text-zinc-400 hover:bg-zinc-800"
    title={copied ? "Copied!" : "Copy response"}
  >
    {copied ? <Check size={14} className="text-green-500" /> : <Copy size={14} />}
  </button>
  {/* ReactMarkdown content */}
</div>
```

**Behavior:** `navigator.clipboard.writeText(message.content)` — copies the raw Markdown string (pre-render text). The `message.content` field holds the full model response text with inline `[Sn]` markers intact.

**Import additions required in MessageBubble.tsx:** `Copy, Check` from `lucide-react`.

**Accessibility:** `title` attribute toggles between "Copy response" and "Copied!" for screen reader feedback. No `aria-label` duplication needed given `title` is present.

## Interaction Contracts

### Post-stream render gate (inherited from Phase 3 D-08)

Nothing in the post-stream section (knob row, follow-up rail, coverage indicator, citation pills) appears until `message.citations !== undefined && !message.isStreaming`. This is an invariant — do not relax it.

### Streaming cursor behavior

The `▋` streaming cursor is rendered via CSS `after:` pseudo-element on the **outer bubble wrapper div** (not inside the ReactMarkdown content). This prevents the cursor from appearing inside a `<p>` or `<li>` rendered by react-markdown, which could produce invalid HTML nesting.

### Follow-up button click flow

1. User clicks e.g. "Cleaner?"
2. `FollowUpRail.onSubmit("Cleaner?")` fires
3. ChatPage calls modified `handleSubmit("Cleaner?")` — which sets `inputValue` to `"Cleaner?"` then immediately submits, or directly passes the override string to `streamChat` without touching textarea state
4. A new user message `{ role: "user", content: "Cleaner?" }` is appended to messages
5. A new assistant message begins streaming
6. The follow-up rail disappears (it only renders on the latest assistant message, which is now the one currently streaming — `isStreaming: true` suppresses it)

### Knob parsing timing

`parseKnobs(message.content)` is called inside the component render function. It runs only after the post-stream render gate opens (citations received, streaming stopped). At that point `message.content` is the completed string.

### Copy button visibility

Copy button uses `opacity-0 group-hover:opacity-100` on the parent `group`. On touch devices where hover is not available, the button remains hidden. This is acceptable for a single-user local desktop tool.

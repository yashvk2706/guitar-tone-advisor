    Create `frontend/utils/parseKnobs.ts`. The `frontend/utils/` directory may not exist — create it.

    Export the KnobValue interface:
      export interface KnobValue { name: string; value: number; }

    Declare the regex as a module-level const named KNOB_RE (exact pattern from UI-SPEC §4):
      /\b(Bass|Mid|Treble|Gain|Volume|Presence|Reverb|Tone|Drive|Level|Sustain|Output|Bright|High|Low|Delay|Mix|Rate|Depth|Feedback)\s*[=:]\s*(\d+(?:\.\d+)?)/gi

    Export the function:
      export function parseKnobs(text: string): KnobValue[]

    Implementation:
    - Use a `Map<string, number>` keyed by lowercase name to implement last-value-wins.
    - Reset lastIndex to 0 before each call (regex has `g` flag — must reset between calls, or use matchAll).
    - Iterate all matches via `regex.exec(text)` loop or `[...text.matchAll(KNOB_RE)]`.
    - For each match: name = match[1] (as found — already capitalized by the alternation), raw = match[2].
    - parsed = parseFloat(raw); if parsed >= 0 && parsed <= 10, add to map keyed by name.toLowerCase().
    - Convert map to KnobValue[] preserving the name casing from the last match for that key.
    - Return the array. Order is insertion order of last-seen keys (Map preserves insertion order).

    IMPORTANT: The `g` flag on a module-level regex makes `lastIndex` stateful. Either
    use `String.prototype.matchAll(KNOB_RE)` (creates a fresh iterator each call, safe)
    or reset `KNOB_RE.lastIndex = 0` at the top of the function. Using matchAll is preferred.

    This file has no external imports (pure TypeScript, no React). It has no side effects.
  </action>

    Step 1 — Create `frontend/components/RotaryKnob.tsx`:

    Props interface (no default export for the interface):
      interface RotaryKnobProps { name: string; value: number; }
    Default export: function RotaryKnob({ name, value }: RotaryKnobProps)

    SVG geometry computation (implement inside component body):
    - clamp value to [0, 10]: `const safeValue = Math.max(0, Math.min(10, value))`
    - Track arc: start at −135° from 12 o'clock (7 o'clock), end at +135° (5 o'clock)
      - trackStartAngleDeg = −135, trackEndAngleDeg = +135
      - Convert to SVG coords: angleRad = (angleDeg − 90) * Math.PI / 180
        (subtract 90° because SVG 0° is 3 o'clock; −90° rotates origin to 12 o'clock)
      - startX = 40 + 30 * Math.cos((−135 − 90) * Math.PI / 180)  ≈ 18.79
      - startY = 40 + 30 * Math.sin((−135 − 90) * Math.PI / 180)  ≈ 61.21
      - endX   = 40 + 30 * Math.cos((+135 − 90) * Math.PI / 180)  ≈ 61.21
      - endY   = 40 + 30 * Math.sin((+135 − 90) * Math.PI / 180)  ≈ 61.21
      - Track path d: `M ${startX} ${startY} A 30 30 0 1 1 ${endX} ${endY}`
        (largeArcFlag=1 because track sweep is 270° > 180°; sweepFlag=1 for clockwise)

    - Value arc: sweep = (safeValue / 10) * 270 degrees
      - If safeValue === 0: render no value arc (or a zero-length arc at the start point only)
      - valueEndAngleDeg = −135 + (safeValue / 10) * 270
      - valueEndX = 40 + 30 * Math.cos((valueEndAngleDeg − 90) * Math.PI / 180)
      - valueEndY = 40 + 30 * Math.sin((valueEndAngleDeg − 90) * Math.PI / 180)
      - sweepAngle = (safeValue / 10) * 270
      - largeArcFlag = sweepAngle > 180 ? 1 : 0
      - Value path d: `M ${startX} ${startY} A 30 30 0 ${largeArcFlag} 1 ${valueEndX} ${valueEndY}`
        (sweepFlag=1 for clockwise, same as track)

    SVG markup:
      <svg viewBox="0 0 80 88" width="80" height="88" aria-label={`${name}: ${value}`}>
        {/* Track arc */}
        <path d={trackD} className="stroke-zinc-700" strokeWidth={5} strokeLinecap="round" fill="none" />
        {/* Value arc — only render if safeValue > 0 */}
        {safeValue > 0 && (
          <path d={valueD} className="stroke-zinc-300" strokeWidth={5} strokeLinecap="round" fill="none" />
        )}
        {/* Center indicator dot */}
        <circle cx="40" cy="40" r="3" className="fill-zinc-400" stroke="none" />
        {/* Value label (numeric) */}
        <text x="40" y="58" textAnchor="middle" fontSize="14" fontWeight="600" className="fill-zinc-300">
          {Number.isInteger(safeValue) ? safeValue : safeValue.toFixed(1)}
        </text>
        {/* Name label */}
        <text x="40" y="82" textAnchor="middle" fontSize="10" fontWeight="400" className="fill-zinc-500">
          {name}
        </text>
      </svg>

    NOTE: Tailwind classes on SVG elements use `className` just like HTML elements in React/JSX.
    Do NOT use inline `style` for colors — use className with Tailwind stroke-* and fill-* utilities.
    The `stroke` and `fill` attributes on the elements must be set via `stroke="none"` / `fill="none"`
    as SVG attributes (not Tailwind) for elements that need no stroke/fill, to avoid SVG default black.
